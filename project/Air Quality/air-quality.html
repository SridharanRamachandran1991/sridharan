Air Quality & Pollution Analysis in India

Comprehensive analysis of historical air quality data (1987–2018) identifying pollution trends, regional hotspots, and temporal patterns across Indian cities and states.

Executive Summary

Historical analysis • 400,000+ records • Multi-pollutant assessment

Dataset scale: Over 400,000 rows of air quality measurements covering pollutants (SO₂, NO₂, RSPM) across multiple cities and states from 1987–2018
Analytical complexity: Handled large-scale time-series data with missing values, inconsistent location names, and required advanced data cleaning and feature engineering
Key insights: Identified regional pollution hotspots, temporal trends, and correlations between pollutants and geographic factors
Technical approach: Python (pandas) for data cleaning and transformation, SQL for aggregation and querying, Tableau for interactive visualizations
Impact: Provides actionable insights for environmental policy and public health initiatives by identifying high-risk areas and pollution trends over time.

Project Overview

This project involves analyzing historical air quality data in India to identify pollution trends, regional hotspots, and correlations with factors like location and time. The complexity stems from handling over 400,000 rows of data, managing missing values, and performing time-series analysis across multiple pollutants and geographic regions.

What I Built

  • Data cleaning pipeline using Python (pandas) to handle missing values, standardize location names, and remove outliers
  • Feature engineering to calculate average pollution levels per zone, year-over-year changes, and temporal patterns
  • SQL queries for data aggregation (GROUP BY state and date) and joining with external location data
  • Interactive Tableau dashboards with area charts, scatter plots, and geographic maps highlighting pollution hotspots
  • Time-series analysis to identify long-term trends and seasonal patterns in air quality

Key Questions

  • Which regions and cities show the highest pollution levels over time?
  • What are the temporal trends in pollutant concentrations (SO₂, NO₂, RSPM)?
  • How do pollution levels correlate with geographic factors and time periods?
  • Which monitoring stations show concerning pollution patterns?

Method (High Level)

  • Data Cleaning: Handled missing pollutant readings using imputation (means) and forward-fill for time-series continuity
  • Data Standardization: Standardized inconsistent city/state names and removed outliers in measurement values
  • Feature Engineering: Calculated average pollution levels per zone, year-over-year changes, and temporal aggregations

  • Visualization: Created interactive dashboards with area charts, scatter plots, and geographic maps in Tableau

Methodology & Analysis

Challenge:

Handling over 400,000 rows of air quality data with significant missing values, inconsistent location names, and outliers in measurement values.

Approach:

  • Missing value imputation: Used mean imputation for pollutant readings where appropriate, and forward-fill for time-series continuity to maintain temporal patterns
  • Location standardization: Standardized inconsistent city and state names using string matching and manual mapping to ensure accurate geographic analysis
  • Outlier removal: Identified and removed outliers in measurement values using statistical methods (IQR, z-scores) to ensure data quality
  • Data validation: Implemented validation checks to ensure pollutant readings are within expected ranges and dates are properly formatted

Tools & Techniques:

  • Python (pandas) for data manipulation and cleaning
  • Statistical methods for outlier detection and imputation
  • String processing for location name standardization
  • Time-series handling for temporal data continuity

Key Outcomes

  • Cleaned dataset ready for analysis with minimal data loss while maintaining temporal integrity
  • Standardized location names enabling accurate geographic aggregation and mapping
  • High-quality data foundation supporting reliable trend analysis and visualization

Challenge:

Transforming raw air quality measurements into meaningful analytical features and aggregations for trend analysis and visualization.

Feature Engineering (Python):

  • Zonal averages: Calculated average pollution levels per geographic zone (city, state) to identify regional patterns
  • Year-over-year changes: Computed percentage changes in pollutant concentrations to identify trends and improvements
  • Temporal aggregations: Created monthly, quarterly, and annual averages for time-series analysis
  • Pollutant ratios: Calculated ratios between different pollutants (e.g., NO₂/SO₂) to identify emission source patterns

SQL Queries:

  • State-level aggregation: Used GROUP BY state, date to calculate zonal averages and identify high-pollution states
  • Time-series aggregation: Aggregated data by year and month to enable trend analysis
  • Location enrichment: Used JOIN operations with external location data for deeper geographic insights
  • Station-level analysis: Queried individual monitoring stations to identify outliers and data quality issues

Key Outcomes

  • Created actionable analytical features enabling trend identification and pattern recognition
  • Efficient data aggregation supporting fast dashboard queries and interactive exploration
  • Structured data ready for advanced visualization and stakeholder communication

Challenge:

Creating interactive, stakeholder-friendly visualizations that communicate complex air quality trends and regional patterns effectively.

Dashboard Components:

  • Area charts: Visualized pollutant trends over time (1987–2018) showing long-term patterns and seasonal variations
  • Scatter plots: Compared station-level pollution measurements to identify outliers and high-risk monitoring locations
  • Geographic maps: Used Tableau's built-in geocoding to highlight high-pollution areas on interactive maps
  • Interactive filters: Enabled filtering by specific pollutants (SO₂, NO₂, RSPM), time periods, and geographic regions

Visualization Features:

  • Time-series analysis showing multi-decade pollution trends
  • Geographic heat maps identifying regional pollution hotspots
  • Comparative analysis across different pollutants and locations
  • Interactive exploration enabling drill-down into specific regions or time periods

Key Outcomes

  • Interactive dashboards enabling stakeholder exploration and data-driven decision making
  • Clear visualization of pollution trends and hotspots supporting environmental policy
  • Accessible presentation of complex analytical findings for non-technical audiences

Key Findings & Insights

Regional Patterns:

Analysis revealed distinct regional pollution hotspots, with certain states and cities consistently showing higher pollutant concentrations. Geographic mapping identified areas requiring targeted environmental interventions.

Temporal Trends:

Time-series analysis uncovered long-term trends in air quality, including periods of improvement and deterioration. Year-over-year comparisons highlighted the effectiveness of environmental policies and identified periods requiring attention.

Pollutant Correlations:

Analysis of multiple pollutants (SO₂, NO₂, RSPM) revealed correlations between different emission sources and geographic factors, providing insights into pollution sources and their regional distribution.

Business Recommendations

  • Prioritize environmental interventions in identified high-pollution regions based on geographic analysis
  • Monitor long-term trends to evaluate the effectiveness of air quality improvement initiatives
  • Use station-level comparisons to identify data quality issues and optimize monitoring network coverage
  • Leverage temporal patterns to predict seasonal pollution spikes and plan preventive measures

Limitations & Next Steps

  • Data completeness: Missing values required imputation, which may introduce uncertainty in trend analysis for specific time periods or locations.
  • Temporal coverage: While the dataset spans 1987–2018, data availability varies by location and time period, affecting comparability.
  • Station representation: Monitoring station density varies by region, potentially underrepresenting pollution levels in areas with fewer stations.

Next Steps

  • Integrate external data sources (weather, industrial activity) to enhance correlation analysis and identify pollution drivers
  • Develop predictive models to forecast pollution levels based on historical trends and external factors
  • Create automated reporting dashboards for real-time monitoring and alerting on pollution threshold breaches
  • Expand analysis to include additional pollutants and health impact assessments
